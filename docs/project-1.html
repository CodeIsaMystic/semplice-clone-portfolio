"Git" A Workflow

Ce projet d'étude a lié la conception du Design, la communication client, un travail simple, propre et "scalable", un workflow automatisé.

Objectifs

Prendre en charge un projet utilisant les méthodes, outils et concepts professionnels.

Stack

HTML5, CSS3, Vanilla Javascript

NPM, Git, Github

WebPack, Babel, PostCSS, Autoprefixer

Topics

BEM, Atomic Design

Files Architecture, Automation

SEO, Accessibility

Problème

Devant la difficulté de formation en France, l'approche de développement web reste très sommaire. Il a été, sur ce projet, un réel défi de produire seul usant d'un workflow automatisé, de concepts UI/UX modernes et de prendre en main des outils & plugins offrant ainsi une approche très professionnelle du métier de développeur web.

La première approche du "web development" est assez simple, l'intégration de ses problématiques est tout autre. Le métier de développeur nous impose une reflexion en profondeur sur les différentes problématiques que l'on pourrait rencontrer mais également l'apprentissage de concepts fondamentaux par l'expérience... Ce projet expose la démarche frontend, tentant ainsi d'aborder le travail en amont avec le client et les équipes "Design" jusque plus tard vers le backend et les équipes "Dev Ops".

Seul sur le projet, la réflexion d'une basique charte graphique s'est vue très enrichissante, m'accompagnant vers les concepts de "Design System", d'Atomic Design, l'utilisation de Figma... Une certaine appétence pour le Design m'a pousser à aller chercher plus loin, m'aidant ainsi à pouvoir aujourd'hui, comprendre les problématiques des équpes travaillant en amont, des utilisateurs et celles des développeurs.

Le Développeur Frontend est impacté par une certaine contradiction ne lui imposant pas le travail sur le Design mais avec les équipes de Web designer. Il reste une approche censée que de comprendre ces problématiques et de pouvoir y répondre...

Comment l'approche du design peut impacter l'ensemble d'un projet web ?

Depuis la "B.E.M   Methodology" vers l' Atomic Design

Le "Naming" de classes CSS, la conception des patterns, la visualisation de Blocks, d'Elements, la différenciation avec les Modifiers. Cette méthodologie m'a clairement aidé à mettre du sens à mes projets, que ce soit par la suite, pour l'organisation de mon architecture de fichiers, pour la conception de Modules Javascript, ou bien la création de "React Components"...

Suivant toujours ce concept "d'encapsulation", j'ai pris le parti dans ce projet de développer les scripts en suivant le paradigme "OOP" même si je suis aujourd'hui plus orienté vers le "Functional Programming", il faut avouer que javascript reste un language orienté "Objet".
